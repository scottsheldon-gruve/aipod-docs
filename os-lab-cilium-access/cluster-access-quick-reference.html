Mac – /etc/hosts
2.1Add entries to /etc/hosts on MacmacOS

Open Terminal and run:

$ sudo nano /etc/hosts

Add these lines at the bottom of the file:

# os-lab-cilium OpenShift cluster 10.7.1.160 api.os-lab-cilium.oslab.oslab 10.7.1.159 console-openshift-console.apps.os-lab-cilium.oslab.oslab 10.7.1.159 oauth-openshift.apps.os-lab-cilium.oslab.oslab 10.7.1.159 hubble.apps.os-lab-cilium.oslab.oslab

Save with Ctrl+X → Y → Enter, then flush DNS cache:

$ sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder # No output = success

Verify it worked:

$ ping -c 1 api.os-lab-cilium.oslab.oslab PING api.os-lab-cilium.oslab.oslab (10.7.1.160): 56 bytes
⚠
Your Mac must be on the same network as the cluster
You must be connected to the lab network (10.7.1.0/24) for these entries to work. The hosts file only maps names to IPs — your Mac still needs a route to reach those IPs.

Windows – hosts file
3.1Add entries to hosts file on WindowsWindows

The Windows hosts file is at C:\Windows\System32\drivers\etc\hosts and requires admin rights to edit.

⚠
Must run Notepad as Administrator
If you open Notepad normally you will not be able to save the hosts file. You must right-click Notepad and select "Run as administrator" first.

Option A – Notepad (GUI):

# 1. Click Start → search "Notepad" # 2. Right-click Notepad → "Run as administrator" # 3. File → Open → navigate to: C:\Windows\System32\drivers\etc\hosts # 4. Change file filter dropdown from "Text Documents" to "All Files" # 5. Add the lines below at the bottom # 6. File → Save

Option B – PowerShell (faster):

# Open PowerShell as Administrator, then run: > Add-Content -Path "C:\Windows\System32\drivers\etc\hosts" -Value "`n# os-lab-cilium OpenShift cluster" > Add-Content -Path "C:\Windows\System32\drivers\etc\hosts" -Value "10.7.1.160`tapi.os-lab-cilium.oslab.oslab" > Add-Content -Path "C:\Windows\System32\drivers\etc\hosts" -Value "10.7.1.159`tconsole-openshift-console.apps.os-lab-cilium.oslab.oslab" > Add-Content -Path "C:\Windows\System32\drivers\etc\hosts" -Value "10.7.1.159`toauth-openshift.apps.os-lab-cilium.oslab.oslab" > Add-Content -Path "C:\Windows\System32\drivers\etc\hosts" -Value "10.7.1.159`thubble.apps.os-lab-cilium.oslab.oslab"

The entries to add (manually if using Notepad):

# os-lab-cilium OpenShift cluster 10.7.1.160 api.os-lab-cilium.oslab.oslab 10.7.1.159 console-openshift-console.apps.os-lab-cilium.oslab.oslab 10.7.1.159 oauth-openshift.apps.os-lab-cilium.oslab.oslab 10.7.1.159 hubble.apps.os-lab-cilium.oslab.oslab
3.2Flush DNS cache on WindowsWindows

After saving the hosts file, flush the DNS cache so Windows picks up the new entries immediately:

# Open Command Prompt or PowerShell as Administrator > ipconfig /flushdns Windows IP Configuration Successfully flushed the DNS Resolver Cache. # Verify it resolves correctly > ping api.os-lab-cilium.oslab.oslab Pinging api.os-lab-cilium.oslab.oslab [10.7.1.160]
